# Requirements Document

## Project Description (Input)

**Issue**: [#5 [MVP] 代理店ダッシュボード（登録＋一覧の最小形・RBAC）](https://github.com/ManatoYamashita/fw-line-meo/issues/5)

- **誰が問題を抱えているか**: 運営（我々）および代理店。現状、両者が担当店舗の状況を一元管理・確認できる場がない。
- **現状**: ダッシュボードが未実装。代理店は担当する飲食店オーナーのオンボーディング（店舗特定＝Place ID・場所・カテゴリの確定）を代行する立場にあるが、それを行うための管理画面が存在せず、登録済み店舗の一覧やステータス（競合設定済みか等）を確認する手段もない。認証・権限分離の仕組みも未整備。
- **何を変えるべきか**: 運営・代理店が同一ダッシュボードに Google ログインでログインし、RBAC でロール分離（運営＝全店舗閲覧、代理店＝担当店のみ）された上で、(1) 担当店の登録（店名検索 → Place 特定 → 基本情報入力）と (2) 担当店の一覧表示（登録済み／競合設定済み等のステータス確認）ができる最小限のダッシュボードを実装する。

### スコープ確認（要件定義時のユーザー回答）

1. **オーナー紐付け**: 店舗登録は「LINE 登録済み（招待コード入力済み）のオーナーを選択して店舗特定を代行」する方式。オーナー未登録での店舗先行登録は行わない。
2. **運営の権限**: 運営ロールも店舗登録を行える（登録時に対象の代理店を指定）。
3. **招待コード発行 UI**: 本 spec に含める（line-onboarding からの先送り分）。
4. **アカウント管理**: 運営による代理店・ダッシュボード利用者の管理 UI を本 spec に含める。

## Introduction

本仕様は、運営と代理店が同一の Web ダッシュボードに Google ログインでログインし、ロールに応じた権限のもとで「担当店舗の登録（オーナーのオンボーディング代行）」「担当店舗の一覧・ステータス確認」「招待コードの発行・管理」「（運営のみ）代理店・利用者アカウントの管理」を行える最小限の管理画面を定義する。代理店は LINE 登録済みのオーナーを選択し、店名検索から Google 上の店舗（Place）を特定して登録することで、IT に不慣れなオーナーに代わって機能1（競合日次サマリー）が稼働開始できる状態を作れる。

以下の受け入れ基準における主語「代理店ダッシュボード」は、本仕様が定義する運営・代理店向け Web 管理画面の機能全体（画面とその背後の要求処理を含む）を指す。

## Boundary Context

- **In scope**: Google ログインによる認証と利用者識別／運営・代理店のロール別アクセス制御（RBAC）／担当店舗の登録（LINE 登録済みオーナーの選択 → 店名検索 → Place 特定 → 基本情報入力 → 確定）／担当店舗の一覧表示（店舗特定・競合設定のステータス含む）／招待コードの発行・無効化 UI／運営による代理店・ダッシュボード利用者の登録・無効化
- **Out of scope**: 店ごとの詳細分析画面・売上レポート（第2フェーズ）／競合店舗の選定・編集（自動選定は日次バッチ側の責務。本ダッシュボードは状態の表示のみ）／オーナーの LINE 側オンボーディングフロー自体（spec: `line-onboarding`）／オーナー未登録のままの店舗先行登録／Google OAuth 連携・機能2（第2フェーズ）／収益・課金管理
- **Adjacent expectations**: 招待コードの検証・消費（オーナーの代理店紐付け）は `line-onboarding` が担い、本ダッシュボードはコードの発行・無効化と状態表示までを担う。競合店舗の自動設定と日次指標の取得は `competitive-daily-summary`（日次バッチ）が担い、本ダッシュボードはその結果を表示するのみで変更しない。Google ログインの認証基盤は `gcp-infra-foundation` が提供済みであることを前提とする。店舗登録の対象オーナーは、`line-onboarding` の招待コード入力により代理店へ紐付け済みであることを前提とする。

## Requirements

### Requirement 1: Google ログインによる認証と利用者識別

**Objective:** As a 運営・代理店, I want 自分の Google アカウントだけでダッシュボードに安全にログインできること, so that パスワード管理の負担や漏えいリスクなしに管理業務を開始できる

#### Acceptance Criteria

1. When 未認証の利用者がダッシュボードのいずれかの画面にアクセスしたとき, the 代理店ダッシュボード shall 管理情報を一切表示せず、Google ログインへの導線を提示する
2. When 利用者が Google ログインに成功し、登録済みのダッシュボード利用者と一致したとき, the 代理店ダッシュボード shall 当該利用者のロールに応じた機能を提示する
3. If Google ログインに成功したが登録済みのダッシュボード利用者と一致しないとき, the 代理店ダッシュボード shall 管理情報を一切表示せず、利用資格がない旨と運営への問い合わせ方法を提示する
4. When 利用者がログアウト操作を行ったとき, the 代理店ダッシュボード shall ログイン状態を終了し、未認証の状態に戻す
5. The 代理店ダッシュボード shall 利用者のパスワードを保持せず、認証手段を Google ログインのみとする

### Requirement 2: ロール別アクセス制御（RBAC）

**Objective:** As a 運営, I want 運営と代理店が同一ダッシュボード上でロールに応じた範囲のみ閲覧・操作できること, so that 代理店間の情報漏えいや越権操作を構造的に防げる

#### Acceptance Criteria

1. While 代理店ロールの利用者がログイン中, the 代理店ダッシュボード shall 店舗・オーナー・招待コードに関する表示と操作の対象を自代理店の担当分のみに限定する
2. While 運営ロールの利用者がログイン中, the 代理店ダッシュボード shall 全代理店の店舗・オーナー・招待コードを閲覧可能とする
3. If 利用者が自らの権限範囲外の情報の取得または操作を要求したとき, the 代理店ダッシュボード shall 画面上の導線の有無にかかわらず当該要求を拒否する
4. The 代理店ダッシュボード shall 運営 → 代理店 → オーナー → 店舗の管理構造に反する紐付け（担当外の代理店・オーナーへの店舗登録等）を作成しない

### Requirement 3: 担当店舗の登録（オーナーのオンボーディング代行）

**Objective:** As a 代理店, I want 担当オーナーの店舗特定（店名検索 → Place 確定 → 基本情報入力）を代行できること, so that IT に不慣れなオーナーでも機能1（競合日次サマリー）を使い始められる状態を作れる

#### Acceptance Criteria

1. When 代理店ロールの利用者が店舗登録を開始したとき, the 代理店ダッシュボード shall 自代理店に紐付く登録済みオーナーの中から対象オーナーを選択できるようにする
2. When 運営ロールの利用者が店舗登録を開始したとき, the 代理店ダッシュボード shall 対象の代理店と、当該代理店に紐付くオーナーを指定できるようにする
3. If 対象として選択可能なオーナーが 1 件も存在しないとき, the 代理店ダッシュボード shall その旨と、オーナーが先に LINE で招待コード入力を済ませる必要がある旨を提示する
4. When 対象オーナーの選択後に利用者が店名を入力して検索したとき, the 代理店ダッシュボード shall Google 上の店舗情報から候補を検索し、店名と住所を含む候補一覧（最大 10 件）を選択可能な形式で提示する
5. If 検索結果が 0 件のとき, the 代理店ダッシュボード shall 見つからなかった旨と、表記を変えた再検索の案内を提示する
6. If 店舗候補の検索が外部要因で失敗したとき, the 代理店ダッシュボード shall エラーである旨と再試行の案内を提示する
7. When 利用者が候補一覧から店舗を選択したとき, the 代理店ダッシュボード shall 選択された店舗の店名・住所を確認できる形で提示し、基本情報の入力と確定の意思確認を行う
8. When 利用者が確定操作を行ったとき, the 代理店ダッシュボード shall 当該店舗を Google 上の一意な店舗（Place）と紐付いた確定状態で対象オーナーの店舗として登録する
9. If 選択された店舗（Place）が既に他の店舗として登録済みのとき, the 代理店ダッシュボード shall 登録を行わず、その旨を提示する
10. When 店舗特定前のオーナーに対する店舗登録が完了したとき, the 代理店ダッシュボード shall 当該オーナーの状態を「店舗特定済み」に遷移させる
11. The 代理店ダッシュボード shall 店舗候補の取得に Google が公式に提供する手段のみを用いる（Web 画面の自動収集等は行わない）

### Requirement 4: 担当店舗の一覧表示

**Objective:** As a 運営・代理店, I want 担当店舗の登録状況と競合設定状況を一覧で確認できること, so that 各店舗がサービスを利用開始できる状態にあるかを一目で把握できる

#### Acceptance Criteria

1. When 代理店ロールの利用者が店舗一覧を表示したとき, the 代理店ダッシュボード shall 自代理店の担当店舗のみを店名とともに一覧表示する
2. When 運営ロールの利用者が店舗一覧を表示したとき, the 代理店ダッシュボード shall 全店舗を担当代理店が識別できる形で一覧表示する
3. The 代理店ダッシュボード shall 一覧の各店舗について、店舗特定の状態（Google 上の店舗と確定済みか）と競合設定の状態（競合店舗が設定済みか）を表示する
4. If 表示対象の店舗が 0 件のとき, the 代理店ダッシュボード shall 0 件である旨と店舗登録への導線を提示する
5. The 代理店ダッシュボード shall 競合設定の内容を変更する手段を提供しない（表示のみとする）

### Requirement 5: 招待コードの発行・管理

**Objective:** As a 代理店, I want 自代理店の招待コードを発行・無効化・確認できること, so that 新しいオーナーを LINE オンボーディングへ自力で案内できる

#### Acceptance Criteria

1. When 代理店ロールの利用者が招待コード管理を表示したとき, the 代理店ダッシュボード shall 自代理店の招待コードを有効・無効の別とともに表示する
2. When 利用者が招待コードの発行操作を行ったとき, the 代理店ダッシュボード shall 対象代理店の新しい招待コードを発行し、オーナーに案内できる形で提示する
3. When 利用者が招待コードの無効化操作を行ったとき, the 代理店ダッシュボード shall 当該コードを無効として記録し、以後のオーナー紐付けに利用できない状態にする
4. While 運営ロールの利用者がログイン中, the 代理店ダッシュボード shall 全代理店の招待コードの閲覧・発行・無効化を可能とする

### Requirement 6: 代理店・ダッシュボード利用者の管理（運営）

**Objective:** As a 運営, I want 代理店とダッシュボード利用者をダッシュボード上で登録・管理できること, so that 新しい代理店の受け入れを手作業のデータ投入なしに完結できる

#### Acceptance Criteria

1. When 運営ロールの利用者が代理店の作成を行ったとき, the 代理店ダッシュボード shall 当該代理店を登録し、代理店の一覧に反映する
2. When 運営ロールの利用者がダッシュボード利用者を登録したとき, the 代理店ダッシュボード shall ロール（運営・代理店）の指定を受け付け、当該利用者が自身の Google アカウントでログインできる状態にする
3. The 代理店ダッシュボード shall 代理店ロールの利用者を必ずいずれか 1 つの代理店に所属させる
4. When 運営ロールの利用者がダッシュボード利用者を無効化したとき, the 代理店ダッシュボード shall 以後の当該利用者によるログインを拒否する
5. If 代理店ロールの利用者が代理店・利用者の管理機能へアクセスしたとき, the 代理店ダッシュボード shall 当該アクセスを拒否する

### Requirement 7: セキュリティ・プライバシー・コンプライアンス

**Objective:** As a 運営, I want 不正アクセス・情報漏えい・規約違反が構造的に防がれること, so that テナント全体（全代理店・全オーナー）を危険に晒さない

#### Acceptance Criteria

1. The 代理店ダッシュボード shall 認証されていない要求に対して店舗・オーナー・代理店・利用者・招待コードのいかなる情報も返さない
2. The 代理店ダッシュボード shall 来店客に関する情報を一切扱わない
3. The 代理店ダッシュボード shall すべての画面・案内文を日本語で提供する
4. If 利用者の操作を処理できない内部障害が発生したとき, the 代理店ダッシュボード shall エラーである旨を提示し、操作が完了したかのような表示を行わない
