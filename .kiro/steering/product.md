# Product Overview

**fw-line-meo（LINE Restaurant Manager）** = 飲食店向け LINE 一元管理アプリ。
ITに不慣れな飲食店オーナーが **LINE だけで** 市場ポジション把握・Google クチコミ獲得促進・（将来）Google ビジネスプロフィール投稿を完結できるサービス。

中核思想は「**オーナー向けに複雑な Web UI を持たず、LINE で全てを一元管理する**」。
この「複雑な Web UI を持たない」原則は **オーナーにのみ** 適用され、代理店・運営には Web ダッシュボードを提供する。

## Core Capabilities

- **機能3（MVP 最優先）**: 口コミ用 QR・タップ式アンケート → Gemini が口コミ下書きを生成。客がコピペして Google へ投稿。
- **機能1（MVP）**: 競合ポジショニング日次サマリーを毎朝 LINE に Flex Message 配信。
- **代理店ダッシュボード（MVP 最小形）**: 店舗登録＋一覧。運営/代理店が RBAC でロール分離して利用。
- **段階的オンボーディング（MVP）**: LINE 上で店舗を特定するまでを誘導。
- **第2フェーズ**: 機能2（GBP 投稿作成・OAuth）、機能1-b（クチコミ返信）。MVP には持ち込まない。

## Target Use Cases

- 多忙で IT リテラシーが高くない飲食店オーナーが、LINE のチャット＋リッチメニューだけで自店の市場状況を把握する。
- 来店客がテーブルの QR を読み、LINE を経由せず通常 Web でアンケートに回答し、Google クチコミ投稿まで摩擦なく到達する。
- 代理店が担当店を Web ダッシュボードに登録し、運営が全店を俯瞰する。

## Value Proposition

- **窓口を LINE に限定**し学習・運用コストを排除（オーナーは日常的に LINE を使える）。
- **客を LINE に登録させない**（機能3 は LINE 非経由）ことで来店客側の摩擦をゼロにする。
- AI 下書きは **客本人が選んだ事実のみ** を反映し、客ごとに語彙を変えてスパム判定を回避する正攻法。

## 侵してはならない制約（プロダクト境界）

これらはプロダクトの存在意義に直結する不可侵の前提。コードはこれに反してはならない。

- **レビューゲーティング禁止**: 高評価のみ Google 誘導・低評価を隠す導線は Google 規約違反。低評価客も同一導線で誘導する。
- **代理投稿禁止**: Google クチコミは客本人が投稿。システムの責務は「下書き生成 → 客がコピペ」まで。
- **スクレイピング禁止**: 競合データは Google Places API のみ（従量課金を許容）。
- **客の個人情報を一切取得しない**: アンケート回答は Place 単位の匿名集計のみ保持。個別回答は永続保存しない。
- **AI ガードレール**: 嘘・誇張・誹謗中傷を生成しない（低評価でも節度ある表現）。
- **GBP OAuth は第2フェーズ**: 審査リスクが高く MVP の生死を賭けない。
